# Portfolio Website Implementation Plan

The goal is to build a professional, responsive, multi-page portfolio for a Management Information Systems student and Communication/Marketing Manager.

## Proposed Changes
We will create a new directory at `C:/Users/Elie/Desktop/aimable/portfolio` (or work within the chosen workspace) containing:

### Core Files
- `index.html`: The landing page with a hero section, brief overview of skills, Christian identity intro, and current role.
- `about.html`: Detailed background, interests, and dedicated Christian journey section.
- `skills.html`: Comprehensive breakdown of skills (Web, Office, Digital Comm, Survey, AI).
- `experience.html`: Experience details and CV download CTA.
- `projects.html`: Showcase of academic and personal projects.
- `gallery.html`: A dedicated media page.
- `contact.html`: Contact form and links.

### Assets
- `css/style.css`: Custom styling, CSS variables for the color palette (`#132231` for primary/text, `#4cc9d3` for accents/buttons, `#cbe7ea` for backgrounds/highlights, `#5d8a9e` for secondary elements), typography imported from Google Fonts, and animations.
- `images/`: Directory to be populated by the user with their own assets.

## Frameworks
- **HTML5** (semantic structure)
- **CSS3** (custom styles, animations)
- **Bootstrap 5** (via CDN for layout, responsiveness, and components like Navbar/Cards)

## Verification Plan
### Manual Verification
- Open the generated `index.html` in the browser to ensure all CSS styles, Bootstrap utilities, and Google Fonts are applied correctly.
- Verify navigation links work across the 7 pages.
- Ensure the specific Christian identity and Choir (`LA LUMIÈRE`) links are properly integrated.
- Ensure mobile responsiveness logic holds up by resizing the browser.
